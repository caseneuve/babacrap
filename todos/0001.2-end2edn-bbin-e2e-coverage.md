## Context

The bbin-installable CLI needs contract-level E2E coverage that exercises the command users actually run after installation. Existing tests are mostly unit/dispatcher-style and do not comprehensively cover all subcommands and flags through a local bbin install.

Use `/home/piotr/git/end2edn` as the E2E framework and run the suite in Podman against a temporary local `bbin install . --as babacrap-dev` binary.

## Acceptance Criteria

- [ ] `end2edn` is added as a pinned test dependency or otherwise invoked reproducibly.
- [ ] A `bb test:e2e` task runs the E2E suite in Podman.
- [ ] The E2E setup installs the current checkout with local bbin as `babacrap-dev` into a temporary bin directory.
- [ ] All E2E cases invoke the installed binary, not direct namespaces or existing `bb` tasks.
- [ ] Fixture/example projects exist for CRAP, mutation, and detangle scenarios.
- [ ] Dispatcher coverage includes no args, `-h`, `--help`, `help`, unknown subcommand, subcommand dispatch, subcommand help, exit-code preservation, and argument forwarding.
- [ ] `crap` E2E coverage includes every public flag: `-p`/`--src`, repeated `--src`, `-s`/`--test`, repeated `--test`, `-n`/`--ns-regex`, `-t`/`--test-ns-regex`, `-o`/`--output`, `--format`, `--crap-threshold`, `--coverage`/`--no-coverage`, `-h`, and `--help`.
- [ ] `mutate` E2E coverage includes every public flag: `-p`/`--src`, repeated `--src`, `-c`/`--test-command`, `--timeout-ms`, `--limit`, `--format`, `--force`, `-h`, and `--help`.
- [ ] `detangle` E2E coverage includes every public flag: `-p`/`--src`, repeated `--src`, file-path `--src`, `--format`, `-h`, and `--help`.
- [ ] Error-path coverage includes unknown options, invalid enum/numeric values, unknown subcommands, and non-zero underlying subcommand exits.
- [ ] Mutation E2E verifies target files are restored and no `*.babacrap.bak` files remain after runs.
- [ ] `bb test`, `bb lint`, `bb dry`, and `bb test:e2e` pass.

## Affected Files

- `bb.edn` — add E2E dependency/task and integrate with test flow if appropriate.
- `test/e2e/cases.edn` or `test/e2e/*.cases.edn` — end2edn suites.
- `test/e2e/fixtures/` or `test/fixtures/e2e/` — generated or template fixture projects.
- `README.md` — document E2E test command and Podman requirement if exposed to contributors.
- `AGENTS.md` — update required checks if `bb test:e2e` becomes canonical.

## E2E Spec

GIVEN a clean checkout mounted into a Podman container
AND local bbin is mounted from `/home/piotr/git/external/bbin`
WHEN the test setup runs `bbin install /work --as babacrap-dev` into a temporary bin directory
THEN the installed `babacrap-dev` executable is available only in that temporary location.

GIVEN the installed `babacrap-dev` executable
WHEN each top-level dispatcher/help/error scenario is run
THEN stdout, stderr, and exit codes match the public CLI contract.

GIVEN fixture projects for CRAP, mutation, and detangle
WHEN each subcommand is run with every documented flag and representative valid/invalid values
THEN outputs, created files, restoration behavior, and exit codes match the public CLI contract.

## Notes

This task is blocked on `0001.1` so the suite can reuse a proven Podman/local-bbin install command.

Keep mutation fixtures tiny and bounded with `--limit` and short `--timeout-ms` values, since `end2edn` currently does not provide per-command run timeouts for `:when :run`.
