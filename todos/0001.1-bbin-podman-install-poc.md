## Context

Before building a comprehensive `end2edn` E2E suite for the bbin-installed CLI, prove the container execution model is reliable and reproducible.

The intended model is to run tests in Podman, install the current checkout with the local `/home/piotr/git/external/bbin` checkout, and execute only the temporary installed binary. This should avoid host/global bbin installs and catch packaging problems that direct `bb -m` tests miss.

## Acceptance Criteria

- [ ] A documented Podman command installs `/work` with local bbin as `babacrap-dev` into a temporary container bin directory.
- [ ] The POC runs `/tmp/bin/babacrap-dev --help` successfully inside the container.
- [ ] The POC runs at least one real subcommand through the installed binary inside the container.
- [ ] The command uses a PATH-safe/temp install location and never installs a development checkout as `babacrap`.
- [ ] Network/cache assumptions are identified explicitly: either the command works offline with mounted caches, or the required network/cache mounts are documented.
- [ ] The resulting command shape is ready to reuse from a `bb test:e2e` task.

## Affected Files

- `bb.edn` — possible temporary/proposed task shape for the POC.
- `todos/0001.1-bbin-podman-install-poc.md` — record the final command and assumptions.

## Notes

Use the local bbin checkout from current master, mounted into the container, rather than installing bbin in the container.

Candidate shape:

```sh
podman run --rm \
  -v "$PWD:/work:ro" \
  -v "/home/piotr/git/external/bbin:/bbin:ro" \
  --tmpfs /tmp:rw,mode=1777 \
  -e END2EDN_SAFE_ENV=1 \
  -w /work \
  docker.io/babashka/babashka \
  sh -lc 'mkdir -p /tmp/bin && BABASHKA_BBIN_BIN_DIR=/tmp/bin /bbin/bbin install /work --as babacrap-dev && /tmp/bin/babacrap-dev --help'
```
